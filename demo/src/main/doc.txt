JWT, token

token base authentication ra đời để hỗ trợ việc xác thực nhanh gọn bớt rườm rà

JWT: header, payload, signature.
    -> payload k nên chứa thông tin quá nhạy cảm, chỉ nên gồm như: obj, role, id, ...
    signature là hash của header + payload + salt secret

    salt: ký (sign) và xác minh (verify) token.
        -> kết hợp với secret key để tạo ra signature bằng thuật toán như HMAC SHA256 (giúp token ko bị sửa, token đc tạo hợp lệ bởi server)
        ->Khi client gửi JWT lên server: - Server sẽ dùng lại secret key đó, - Tự tính lại signature từ header + payload, - So sánh với signature trong token
